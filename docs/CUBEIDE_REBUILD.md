# Rebuilding in STM32CubeIDE

CubeIDE **does not generate** `link.c`, `fan.c`, `setpoint.c`, or `usart.c` — those are custom files already under `Core/Src/`. A normal **Build** rescans that folder and regenerates `Debug/objects.list`, `subdir.mk`, and the makefile.

## Path A — Refresh build files only (recommended first)

Use this when you only need makefiles updated to include the new `.c` files.

1. **Open the project**  
   File → Open Projects from File System → select the `Heatpump` folder (the one containing `heatpump_ctrl.ioc`).

2. **Refresh**  
   Right-click project `heatpump_ctrl` → **Refresh** (or F5).

3. **Confirm custom sources appear** in Project Explorer:
   - `Core/Src/link.c`
   - `Core/Src/fan.c`
   - `Core/Src/setpoint.c`
   - `Core/Src/usart.c`

   If any are missing, use **Import** → General → File System and import them into `Core/Src`.

4. **Clean and build**  
   Project → **Clean…** → Clean all projects → OK  
   Project → **Build All** (Ctrl+B)

5. **Check the build log** includes:
   - `link.c`, `fan.c`, `setpoint.c`, `usart.c`
   - `stm32g4xx_hal_uart.c` (HAL UART must stay enabled — see below)

6. **If UART link errors appear** after Cube regenerated `stm32g4xx_hal_conf.h`:
   - Open `Core/Inc/stm32g4xx_hal_conf.h`
   - Ensure these lines are **uncommented**:
     ```c
     #define HAL_UART_MODULE_ENABLED
     ```
   - Build again.

7. **Verify `main.c` USER CODE** (Cube must not remove this):
   ```c
   /* USER CODE BEGIN 2 */
   App_Init();
   /* USER CODE END 2 */
   ```
   And keep the note that `HAL_TIM_PeriodElapsedCallback` lives in `app.c`, not `main.c`.

## Path B — Regenerate from CubeMX (.ioc) + USART1

Use this if you want USART1 (dashboard serial) defined in CubeMX on **PA9 / PA10**.

### B1. Enable USART in CubeMX

1. Open `heatpump_ctrl.ioc` (double-click in CubeIDE).
2. **Pinout** → click **PA9** → **USART1_TX**, **PA10** → **USART1_RX**.
3. **Connectivity → USART1**:
   - Mode: **Asynchronous**
   - Baud Rate: **115200**
   - Word Length: 8, no parity, 1 stop bit
4. **Project Manager** tab:
   - **Keep User Code** = checked (already set in your `.ioc`)
   - **Generate peripheral initialization as pair** = as you prefer
5. **Middleware / ADC / etc.** — leave existing config unchanged.

### B2. Generate code

1. Click **GENERATE CODE** (or Ctrl+S on .ioc then generate).
2. When prompted, choose **Yes** to merge / keep user code.

### B3. Resolve USART duplicates (important)

Cube will create its own `Core/Src/usart.c` with `MX_USART1_UART_Init()` and put `HAL_UART_MspInit()` in `stm32g4xx_hal_msp.c`.

You already have a manual `usart.c` with the same symbols. **Pick one approach:**

**Option 1 — Keep Cube’s USART (cleanest long-term)**

1. Delete or rename the old manual `Core/Src/usart.c` if Cube generated a new one (compare files).
2. Ensure `link.c` still has `#include "usart.h"` and uses `huart1`.
3. Remove any second `HAL_UART_MspInit` — only the copy in `stm32g4xx_hal_msp.c` should remain.

**Option 2 — Keep manual `usart.c` (current tree)**

1. In Cube **disable** USART1 again *or* after generate, remove Cube’s duplicate `MX_USART1_UART_Init` from generated `usart.c` and keep your file.
2. Do **not** add `MX_USART1_UART_Init()` to `main.c` if `Link_Init()` already calls it inside `App_Init()`.

### B4. HAL and main.c after generate

1. `stm32g4xx_hal_conf.h` — confirm `HAL_UART_MODULE_ENABLED` is set.
2. `main.c` — Cube may insert `MX_USART1_UART_Init();` before USER CODE. Either:
   - Leave it **and** remove `MX_USART1_UART_Init()` from `Link_Init()`, **or**
   - Delete the line from `main.c` and keep init only in `Link_Init()` (current design).
3. Re-apply if Cube overwrote `app.c` integration — `App_Init()` must still call `Fan_Init()`, `Link_Init()`, and `Ms100Task` must call `Link_Process()`.

### B5. Build

Project → **Clean** → **Build All**.

## What Cube regenerates vs what you keep

| Regenerated by Cube | Your custom files (never auto-deleted) |
|---------------------|--------------------------------------|
| `main.c` (USER CODE preserved) | `app.c`, `link.c`, `fan.c`, `setpoint.c` |
| `gpio.c`, `adc.c`, `tim.c`, … | `thermal_control.c`, `valves.c`, `can_comp.c`, … |
| `stm32g4xx_hal_msp.c` | `usart.c` (unless you adopt Cube’s) |
| `Debug/makefile`, `objects.list`, `subdir.mk` | `dashboard/` (separate) |

## Quick checklist after any rebuild

- [ ] Build finishes with **0 errors**
- [ ] `heatpump_ctrl.elf` updated in `Debug/`
- [ ] Flash firmware; connect USB-UART to **PA9/PA10** @ 115200
- [ ] Run dashboard bridge: `python dashboard/bridge/server.py --port <your-port>`

## If custom files disappear from the build

CubeIDE only compiles sources under `Core/` and `Drivers/`. If `objects.list` omits a file:

1. Project → **Properties** → **C/C++ Build** → **Settings** — confirm toolchain is active.
2. **Clean** + **Build All** again.
3. As last resort, delete the `Debug` folder (not `Core`), then **Build All** — CubeIDE recreates `Debug` from scratch.

Do **not** delete `Core/Src` custom files.
